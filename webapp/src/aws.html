AWS DevOps Steps Overview

SCM - Github

Load files into github

 

CodeDeploy

Application -> create application

-> name - app name

-> compute platform - Amazon EC2/On-prem

 

Deployment Group (target to which instance to deploy the application too)

 

  • Before creating a DG you need to have an EC2 instance up and running
    • Spin EC2 instance -> AMI Linux 2 AMI (HVM) - t2.micro
      • IAM Role -> attach a role (EC2 code deploy)
        • Create IAM role -> permission -> amazonec2roleforcodedeploy
      • User Data as text -> enter code for codedeploy agent in redhat
      • Add tag > key - Name , Value - webserver (need tag for codedeploy)
      • Add security group -> allow SSH and HTTP

 

  • Once EC2 instance is up we can create the DG

 

  • Create Codedeploy Deployment Group

-> Deployment group name - webserver-dpeloy

-> Service role -> iam role created earlier for codedeploy-role1

  • IAM create role

-> Deployment Type - In-Place

-> Env configuration - AWS EC2 Instance

Name - WebServer (from the tag on EC2 instance)

 

-> Deployment Settings - CodeDeploy One at a time

-> Loadbalance -> uncheck enable load balancing since we are not using a LB

-> Create Deployment Group

 

Use codepipeline to trigger the deployment instead of triggering from codedeploy every time

 

CodePipeline - setup the pipeline

 

SCM (github) Connect

CodePipeline -> Create a new pipeline

-> Pipeline name - webserver

-> Service Role - create new service role

Check - allow aws code pipeline to create a service role to use with pipeline

-> Advanced settings

  • Artifact store - Default location (creates a S3 bucket)
  • Encryption Key - Default AWS Managed Key

 

-> Add Source Provider

          • Source provider - Github
            • Connect to Github -> using webhook
          • Repository -> github repo name
          • Branch -> github repo branch
          • Change Detect Options -> Github webhooks

 

Build Stage

CodeBuild -> optional if there is nothing to build

 

Deploy Stage

Deploy Provider -> AWS CodeDeply

-> Region - same as codedeploy

-> App name - select codedeploy app name

-> DG - select DG group created earlier

 

Review and Create Pipeline
